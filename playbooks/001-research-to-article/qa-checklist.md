# QA Checklist — Research-to-Article Pipeline

Use this checklist to review article packages generated by the Builder agent.

## 1. Work packet alignment

- [ ] Output matches the approved angle.
- [ ] Output matches the intended audience.
- [ ] Output includes all requested artefacts.
- [ ] Risks and sensitivities from the intake have been considered.
- [ ] Open questions are captured clearly.

## 2. Argument quality

- [ ] Central thesis is clear.
- [ ] Opening creates interest quickly.
- [ ] Argument flows logically.
- [ ] The article does not wander into unrelated territory.
- [ ] The conclusion returns to the main point.
- [ ] Reader can explain the point after one read.

## 3. Audience value

- [ ] Useful for delivery, change, governance, AI adoption, or internal communications audiences.
- [ ] Gives the reader something practical to use.
- [ ] Avoids generic “AI is changing work” commentary.
- [ ] Shows a professional delivery lens.
- [ ] Provides a useful distinction, model, checklist, or decision aid.

## 4. Evidence and claims

- [ ] No unsupported statistics.
- [ ] No named external claims without source support.
- [ ] Speculative claims are framed as likely risks, not facts.
- [ ] Concepts are clearly defined.
- [ ] Terms such as “attention debt” are presented as working concepts unless sourced.
- [ ] Claims are not overstated.

## 5. Tone and voice

- [ ] Practical.
- [ ] Senior.
- [ ] Clear.
- [ ] Calm.
- [ ] Not hype-driven.
- [ ] Not anti-AI.
- [ ] Not smug.
- [ ] Not preachy.
- [ ] Not generic LinkedIn thought-leadership.
- [ ] Sounds like a professional with delivery experience.

## 6. AI-slop risk

Flag if the draft includes:

- [ ] vague transformation language
- [ ] too many rhetorical contrasts
- [ ] repetitive sentence structures
- [ ] obvious “not this, but that” framing
- [ ] empty phrases such as “in today’s fast-paced world”
- [ ] polished but low-value paragraphs
- [ ] overuse of lists of three
- [ ] excessive abstraction without examples

## 7. Structure and readability

- [ ] Strong opening.
- [ ] Short paragraphs.
- [ ] Clear subheadings.
- [ ] Easy to skim.
- [ ] Key point appears early.
- [ ] Practical section appears before reader patience runs out.
- [ ] Ending is sharp enough.
- [ ] LinkedIn post is not too long for the channel.
- [ ] Blog draft is not padded.

## 8. Supporting artefact

- [ ] Artefact is useful standalone.
- [ ] Artefact has a clear use case.
- [ ] Artefact is not too long for practical use.
- [ ] Artefact could become a downloadable/checklist/template asset.
- [ ] Artefact reinforces the article argument.
- [ ] Artefact has a clear title.

## 9. Reputational and professional risk

- [ ] Does not sound like the user is criticising current or past employers.
- [ ] Does not reveal confidential delivery experience.
- [ ] Does not overclaim expertise.
- [ ] Does not imply AI should replace professional judgement.
- [ ] Does not make unsupported claims about organisational behaviour.
- [ ] Would be safe to share with recruiters, clients, peers, or senior stakeholders.

## 10. Approval recommendation

Choose one:

- [ ] Publish-ready
- [ ] Minor revision
- [ ] Major revision
- [ ] Park / reject

Include a short rationale.
